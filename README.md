# launch

Fast entry surface for `qvra`.

`launch` exists to shorten time from visitor to working surface.
It is where one-command beginnings, sharp quickstarts, and low-friction repo entry kits belong.

## What belongs here

- starter commands
- bootstrap scripts
- fast setup surfaces
- low-friction first-run paths
- direct route kits that get a repository moving immediately

## What does not belong here

`launch` is not:
- a generic template graveyard
- a full framework
- a substitute for `kit`
- a repo that requires long reading before use

## Current role in qvra

`launch` is part of the structural spine.
It converts field interest into immediate action.

Related routes:
- [`qvra`](https://github.com/qvra/qvra) — front door
- [`site`](https://github.com/qvra/site) — public web surface
